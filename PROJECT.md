# Dasha — Система биометрической аутентификации и генерации криптографических ключей на основе голоса

**Версия:** 1.0 (май 2026)
**Статус:** Рабочая версия с единой нормализацией

---

## 1. Цель проекта

Создать систему, которая:
- Принимает голосовую запись пользователя
- Извлекает устойчивые биометрические признаки (39-мерный вектор)
- Преобразует их в детерминированный криптографический ключ по ГОСТ Р 52633.0-2006 / 52633.4-2011 / 52633.5-2011
- Не хранит сам голос или биометрический шаблон в открытом виде
- Обеспечивает высокую стабильность ключа при повторных записях одного и того же человека

Система предназначена для дипломной работы и последующей возможной сертификации.

## 2. Текущая архитектура (май 2026)

### 2.1. Общий пайплайн обработки голоса

```mermaid
graph TD
    A[Аудиофайл .wav] --> B[Загрузка + Pre-emphasis]
    B --> C[Детекция речи VAD]
    C --> D[MFCC + Delta + Delta-Delta (39 признаков)]
    D --> E[CMVN нормализация по utterance]
    E --> F[Mean pooling]
    F --> G[MinMax scaling в [0, 1]]
    G --> H[Вектор 39-dim]
    H --> I[НПБК / График в Gradio]
```

### 2.2. Единая нормализация (самое важное)

**Метод:** `CMVN + Mean pooling + MinMax[0,1]`

- **CMVN** — убирает влияние микрофона, громкости и акустической среды
- **Mean pooling** — усредняет признаки по всем активным кадрам
- **MinMax[0,1]** — финальная нормализация, которая выделяет самые характерные коэффициенты именно для этого спикера

**Результат:**
- Все векторы всегда в диапазоне **[0, 1]**
- Один и тот же вектор используется и для визуализации, и для нейросетевого преобразователя
- Хорошая различимость между разными спикерами

## 5. Ключевые решения и почему они приняты

| Решение                    | Обоснование                                                                 |
|----------------------------|-----------------------------------------------------------------------------|
| 39 признаков (MFCC+Δ+Δ²)   | Стандарт в speaker recognition, повышает точность на 8–15%                 |
| CMVN + MinMax[0,1]         | Лучшее сочетание для удаления шума и выделения индивидуальных особенностей  |
| Единый вектор для всего    | Упрощает архитектуру и гарантирует consistency между UI и НПБК            |
| Gradio-интерфейс           | Удобно для демонстрации и отладки во время дипломной работы               |
